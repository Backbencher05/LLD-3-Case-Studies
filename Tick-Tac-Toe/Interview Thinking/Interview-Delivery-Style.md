
# 🎤 Interview Delivery Style

One thing I'd like us to practice throughout this journey is **how we speak** in an interview.

Avoid sounding like you're reading a checklist:

❌

> Board size... Human player... CLI... Database...

Instead, sound conversational and structured:

> "Before designing, I'd like to clarify a few assumptions so that we solve the right problem. I'll assume we're designing the core game engine with an in-memory implementation. The UI isn't the focus, so I'll use a simple CLI while keeping the domain independent of presentation. I'll also keep features like multiplayer and persistence out of scope for the initial version."

Notice the difference.

It feels like a design discussion rather than an exam answer.
