Your Assignment (Very Important)

Pretend I am the Product Manager.

Your task is to ask me at least 20 questions before you agree to build Tic-Tac-Toe.

Don't worry about whether the questions are "good" or "bad." Just think like an engineer trying to remove ambiguity.

# To help you, you can think in these categories:

- Game rules
- Players
- Board
- Winning conditions
- Invalid moves
- User interaction
- Extra features
- Future enhancements


Think like a Software Architect

What questions would you ask the Product Manager?

Examples (don't answer these yet; they're meant to show the type of thinking):

- Is the board always 3×3?
- Can the board size change?
- Is it only for two players?
- Can one player be a computer?
- Should players be able to restart the game?
- Should we keep score across multiple games?
- Is this a console application, web application, or mobile application?
- What happens if someone tries to play on an occupied cell?
- Can players undo a move?
- Should the game detect a draw?
- Who gets the first turn?
- Can players choose their symbols?

Notice something important:

None of these questions are about code.

They're all about understanding the problem.
